# 烟花大师 UI/UX 设计稿

> 版本：v1.1（讨论稿）  
> 范围：MVP 全部 7 个视图 + 首页入口  
> 目标：在保留现有暗色烟花主题的前提下，降低认知负担、增强反馈、提升移动端可用性。

---

## 1. 设计原则

| 原则 | 说明 | 在本游戏中的体现 |
|---|---|---|
| **一次只做一件事** | 每个屏幕只呈现一个核心决策 | Hub 只导航，Build 只编队，Result 只复盘 |
| **状态可见** | 资金、名气、等级、预算等关键信息始终可见或一键可达 | 顶部状态栏常驻；编队页实时预算条 |
| **即时反馈** | 每次点击都有视觉或数字反馈 | 按钮 hover/active；数字变化动画；按钮禁用原因提示 |
| **容错** | 允许撤销、返回、跳过 | 返回按钮常驻；清空节目单；跳过表演 |
| **移动优先** | 单手操作、拇指热区、防止误触 | 底部主操作、大按钮、安全边距 |

## 2. 色彩与字体系统

### 2.1 色彩（在现有基础上细化）

```
--bg:        #0b0d14   页面背景
--panel:     #15182a   卡片/面板背景
--elevated:  #1f2438   按钮/悬停背景
--text:      #f0f0f5   主文字
--muted:     #9aa0b8   次级文字
--accent:    #ffaa33   主强调色（烟花、资金、S级）
--accent2:   #66ccff   次强调色（名气、A级、链接）
--success:   #55cc88   成功/晋升
--danger:    #ff5555   删除/重置
--warn:      #ffcc33   警告/预算紧张
--info:      #aa88ff   提示/标签
```

### 2.2 语义色

| 场景 | 颜色 | 说明 |
|---|---|---|
| 资金 | `--accent` 金黄 | 烟花、金币联想 |
| 名气 | `--accent2` 冰蓝 | 声望、清冷高级感 |
| 主线赛事 | `--info` 紫 | 最重要，最突出 |
| 活动 | `--accent2` 蓝 | 日常、稳定 |
| 重复/保底 | `--muted` 灰/绿 | 可选、安全网 |
| 预算充足 | `--success` | 进度条 > 70% |
| 预算紧张 | `--warn` | 进度条 30%-70% |
| 预算超支 | `--danger` | 进度条 < 30% 或变红 |

### 2.3 字体层级

| 层级 | 尺寸 | 字重 | 用途 |
|---|---|---|---|
| H1 | 40px | 800 | 开始页、结局页大标题 |
| H2 | 24px | 700 | 各视图标题 |
| H3 | 18px | 600 | 面板标题 |
| Body | 14-16px | 400 | 正文、按钮文字 |
| Caption | 12px | 400 | 标签、提示、元信息 |
| Number | 28-48px | 800 | 资金、名气、分数 |

## 3. 全局组件规范

### 3.1 顶部状态栏（Hub / Build / Result / Lab 共享）

```
┌─────────────────────────────────────────┐
│ 🎆 烟花大师          学徒 │ ¥500 │ ⭐0 │
└─────────────────────────────────────────┘
```

- 左侧：品牌Logo + 游戏名（点击返回 Hub）
- 右侧三栏：等级徽章、资金、名气
- 资金/名气变化时：数字颜色闪烁 + 300ms 内滚动到新值

### 3.2 按钮

| 类型 | 样式 | 用途 |
|---|---|---|
| Primary | 金黄背景，深字 | 主操作：新游戏、确认表演 |
| Secondary | 深蓝灰背景，白字 | 次操作：返回、跳过 |
| Success | 绿色背景 | 可晋升、可领取 |
| Danger | 红色背景 | 重置存档、删除 |
| Ghost | 透明+边框 | 低优先级：清空节目单 |

**交互状态**：
- Hover：亮度 +10%
- Active：按下位移 1px
- Disabled：透明度 0.4，cursor:not-allowed，显示 `title` 提示原因
- 主操作按钮宽度最小 120px，高度 44px（拇指热区）

### 3.3 卡片

```
┌────────────────────────────────────┐
│ 赛事名称                    [标签] │  ← 标题行
│ 简介文字...                        │  ← 正文
│ 预算 ¥80 │ 最大6发 │ 奖励 ¥30 ⭐60 │  ← 元信息
│ [选择]                             │  ← 操作
└────────────────────────────────────┘
```

- 主线赛事卡片：左侧 4px 紫色边框
- 活动卡片：左侧 4px 蓝色边框
- 重复/保底：左侧 4px 灰色/绿色边框
- Hover 时整体上移 2px + 阴影

### 3.4 Toast / 临时提示

- 位置：屏幕底部居中
- 背景：`--panel` + 1px border
- 最大宽度 320px，圆角 12px
- 显示 2s 后自动消失
- 用于：研发成功、晋升提示、资金不足提示


**改进点**：
1. **偏好摘要常驻顶部**：让玩家编队时有目标
2. **预算进度条**：实时显示已用/剩余，颜色随紧张度变化
3. **配方按钮高亮**：当配方匹配当前事件偏好时，边框变为强调色
4. **实时预计评分**：在预算条下方显示「预计得分：约 62 分（B）」
5. **清空按钮二次确认**：如果已选 > 0，点击清空时先询问「确定清空？」
6. **确认表演按钮**：未满足条件时 disabled，hover/点击显示原因（预算超支 / 弹数超限 / 未选择）

**移动端适配**：
- 窄屏时「可用配方」和「当前节目单」上下堆叠
- 配方按钮改为两列网格

## 6. 动画与微交互规范

| 场景 | 动画 | 时长 |
|---|---|---|
| 视图切换 | fade + slide 0.2s | 200ms |
| 数字变化 | 颜色闪烁 + 数字滚动 | 300ms |
| 按钮 hover | 亮度提升 + 上移 1px | 150ms |
| 卡片 hover | 上移 2px + 阴影 | 150ms |
| 预算条变化 | 宽度过渡 | 200ms |
| Toast 出现/消失 | 上移 + 淡入/淡出 | 200ms |
| 烟花表演 | Canvas 物理动画 | 实时 |
| 结局庆祝 | 烟花 + 文字缩放脉冲 | 1s 循环 |

---

## 7. 移动端适配要点

1. **安全区**：底部操作栏留出 `env(safe-area-inset-bottom)`
2. **按钮高度**：所有可点击元素最小 44px
3. **字体**：最小 14px，避免缩放
4. **编队页**：窄屏时配方与节目单上下堆叠
5. **表演页**：跳过按钮增加 padding，防止误触系统手势
6. **Hub**：按钮在窄屏单列排列
