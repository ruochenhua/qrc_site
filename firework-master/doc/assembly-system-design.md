# 烟花大师 —— 自由组装系统设计稿

> 版本：v1.0（讨论稿）  
> 范围：MVP 核心组装系统、评分系统、蓝图系统  
> 目标：把游戏核心从「解锁固定配方」改为「解锁材料并自由组装」，让烟花构筑成为游戏主体。

---

## 1. 设计目标

1. **玩家解锁的是材料，不是固定答案**
   - 组件随等级/名气解锁
   - 系统配方仅作为示例蓝图存在
2. **简单规则产生复杂表现**
   - 火药量决定高度和规模
   - 壳体决定形状和容量
   - 颜色剂决定颜色和密度
   - 引线决定爆炸高度
   - 特效剂触发不同视觉效果
3. **支持玩家创造力**
   - 可保存自定义蓝图
   - 多层壳支持二次爆炸
   - 混合颜色、叠加特效
4. **保留赛事经营框架**
   - 事件偏好从离散标签改为连续维度
   - 评分依然基于「玩家设计 vs 事件需求」的匹配度

---

## 2. 核心概念

| 术语 | 含义 |
|---|---|
| **组件（Component）** | 可购买的基础材料：火药、壳体、颜色剂、引线、特效剂 |
| **烟花弹（Shell）** | 一次组装的结果，由组件组合生成 |
| **蓝图（Blueprint）** | 玩家保存的一组组件选择，可一键加载复用 |
| **配方（Recipe）** | 系统自带的固定示例蓝图，免费获得 |
| **壳体容量（Capacity）** | 壳体能容纳的组件总量上限 |
| **多层壳（Multi-break Casing）** | 支持主爆炸 + 二次爆炸的特殊壳体 |
| **属性（Attribute）** | 烟花弹的连续数值：高度、规模、密度、持续时间、颜色向量、特效向量 |

---

## 3. 组件分类与属性

### 3.1 火药（Gunpowder）

决定升空高度、爆炸规模和粒子数。火药越多，烟花飞得越高、爆得越大。

| 型号 | 单位推力 | 单位成本 | 解锁 fame | 描述 |
|---|---|---|---|---|
| 黑火药 | 1.0 | 2 | 0 | 最基础的火药，推力一般，便宜稳定 |
| 细颗粒火药 | 1.5 | 4 | 100 | 颗粒更细，燃烧更快，推力更强 |
| 高性能火药 | 2.2 | 8 | 400 | 专业比赛级火药，能打出高空大花 |
| 军用级火药 | 3.0 | 15 | 1200 | 极限推力，轻松突破高空限制 |

- 火药量 → 高度和爆炸规模正相关
- 火药量过大但壳体容量不足 → 组装失败提示

### 3.2 壳体（Casing）

决定形状、容量、成本，以及是否支持二次爆炸。壳体是烟花弹的容器，所有其他材料都塞在里面。

| 型号 | 形状 | 容量 | 层数 | 成本 | 解锁 fame | 描述 |
|---|---|---|---|---|---|---|
| 小号牡丹壳 | 牡丹 | 8 | 1 | 3 | 0 | 新手常用，球形炸开，小巧 |
| 中号牡丹壳 | 牡丹 | 14 | 1 | 5 | 0 | 标准牡丹壳，容量适中 |
| 小号菊花壳 | 菊花 | 10 | 1 | 4 | 100 | 细长花瓣状炸开 |
| 中号菊花壳 | 菊花 | 16 | 1 | 6 | 100 | 标准菊花壳，花瓣更丰满 |
| 环壳 | 环 | 8 | 1 | 5 | 100 | 炸成空心圆环 |
| 心壳 | 心 | 10 | 1 | 6 | 200 | 炸成爱心形状 |
| 星壳 | 星 | 10 | 1 | 6 | 200 | 炸成五角星形状 |
| 柳壳 | 柳 | 12 | 1 | 7 | 400 | 粒子向下垂落，像柳枝 |
| 棕榈壳 | 棕榈 | 16 | 1 | 8 | 400 | 多条放射臂，像棕榈叶 |
| 小号双层壳 | 牡丹 | 10+4 | 2 | 10 | 400 | 支持主爆后第二次小爆炸 |
| 中号双层壳 | 菊花 | 14+6 | 2 | 14 | 1200 | 双层菊花壳，二次爆炸更华丽 |
| 大号双层壳 | 牡丹 | 18+8 | 2 | 20 | 3000 | 顶级双层壳，压轴专用 |

- 形状直接决定爆炸视觉形态
- 容量限制火药 + 颜色剂 + 特效剂 + 二次装药的总占用
- 多层壳的主层和二层有独立容量
- 二层不允许装火药，只能装二次爆炸药、颜色剂、特效剂

### 3.3 颜色剂（Colorant）

决定粒子颜色和密度。放得越多，烟花越浓密、越鲜艳；可以混合多种颜色。

| 型号 | 颜色 | 密度贡献 | 单位成本 | 解锁 fame | 描述 |
|---|---|---|---|---|---|
| 红剂 | 红 | 1.0 | 1 | 0 | 经典红色，节日气氛 |
| 金剂 | 金 | 1.0 | 1 | 0 | 华丽金色，适合庆典 |
| 蓝剂 | 蓝 | 1.0 | 2 | 100 | 清冷蓝色，专业赛事常用 |
| 绿剂 | 绿 | 1.0 | 2 | 100 | 自然绿色，柳枝必备 |
| 白剂 | 白 | 1.0 | 1 | 0 | 纯净白色，百搭 |
| 紫剂 | 紫 | 1.0 | 3 | 400 | 高贵紫色，浪漫场合 |
| 粉剂 | 粉 | 1.0 | 3 | 400 | 柔和粉色，婚礼常用 |
| 银剂 | 银 | 1.0 | 2 | 1200 | 闪亮银色，高级感 |
| 混色剂 | 彩虹 | 1.0 | 5 | 3000 | 自动循环多种颜色 |

- 颜色剂总量 → 粒子密度
- 多种颜色剂按比例分配粒子颜色
- 颜色剂太少 → 烟花稀疏暗淡
- 颜色剂种类不限制，但每种都占容量

### 3.4 引线（Fuse）

决定爆炸高度。引线越长，烟花飞得越高才爆炸。

| 型号 | 长度 | 爆炸高度系数 | 成本 | 解锁 fame | 描述 |
|---|---|---|---|---|---|
| 短引线 | 短 | 0.6 | 1 | 0 | 低空爆炸，适合近距离观赏 |
| 中引线 | 中 | 1.0 | 1 | 0 | 标准高度，最常用 |
| 长引线 | 长 | 1.4 | 2 | 100 | 高空爆炸，适合大型赛事 |
| 超长引线 | 超长 | 1.8 | 4 | 1200 | 极限高空，压轴专用 |

- 爆炸高度 = 基础高度 × 引线高度系数
- 短引线适合低空表演，长引线适合高空表演

### 3.5 特效剂（Effect Agent）

达到阈值后触发特效。可以叠加多种，无冲突规则。

| 型号 | 特效 | 触发阈值 | 强度/单位 | 成本 | 解锁 fame | 描述 |
|---|---|---|---|---|---|---|
| 闪光粉 | 闪光 | 2 | 0.5 | 2 | 100 | 粒子闪烁，增加华丽感 |
| 爆响药 | 爆响 | 2 | 0.5 | 2 | 0 | 爆炸时发出噼啪声（视觉表现：细碎火花） |
| 尾迹药 | 尾迹 | 2 | 0.5 | 2 | 100 | 上升阶段和粒子拖尾明显 |
| 频闪药 | 频闪 | 2 | 0.5 | 3 | 400 | 粒子明暗交替，节奏感强 |
| 花束药 | 花束 | 3 | 0.4 | 4 | 400 | 爆炸后散落小花团 |
| 二次爆炸药 | 二次爆炸 | 4 | 0.3 | 6 | 400（需多层壳） | 装在多层壳二层，主爆后触发第二波爆炸 |

- 特效强度 = 数量 × 强度/单位
- 多种特效剂可叠加，无冲突规则
- 二次爆炸药只有在多层壳的二层中才能发挥作用

## 4. 组装规则

### 4.1 单发烟花弹的组装输入

```js
{
  gunpowder: { g001: 3 },           // 火药型号: 数量
  casing: 'c001',                    // 壳体型号
  colorant: { col001: 2, col002: 1 }, // 颜色剂型号: 数量
  fuse: 'f001',                      // 引线型号
  effect: { e001: 2, e003: 1 }       // 特效剂型号: 数量
}
```

### 4.2 容量检查

```
总占用 = 火药总数量 + 颜色剂总数量 + 特效剂总数量
总占用 ≤ 壳体容量
```

对于多层壳：

```
主层占用 = 火药 + 颜色剂 + 特效剂（不含二次爆炸药）
二层占用 = 二次爆炸药 + 可选的二层颜色剂
主层占用 ≤ 主层容量
二层占用 ≤ 二层容量
```

### 4.3 组装输出

```js
{
  id: 'shell_xxx',
  name: null, // 或玩家命名
  components: { /* 原始输入 */ },
  height: 0.75,      // 0~1 连续值
  scale: 0.6,        // 0~1 连续值
  density: 0.5,      // 0~1 连续值
  duration: 0.7,     // 0~1 连续值
  color: { red: 0.67, gold: 0.33 }, // 颜色比例向量
  effects: { crackle: 1.0, tail: 0.5 }, // 特效强度向量
  shape: 'peony',
  hasSecondary: false,
  secondary: null,
  cost: 23,
  recipeId: null,
  blueprintId: null
}
```

## 5. 属性计算公式

### 5.1 基础推力

```
thrust = Σ (gunpowder[型号] × 该型号单位推力)
```

### 5.2 高度（Height）

```
baseHeight = normalize(thrust, 0, MAX_THRUST)  // 0~1
fuseFactor = fuse.heightCoefficient             // 0.6~1.8
height = clamp(baseHeight × fuseFactor, 0, 1)
```

### 5.3 规模（Scale）

```
scale = normalize(thrust, 0, MAX_THRUST) × casing.scaleMultiplier
```

壳体的 scaleMultiplier 默认为 1.0，大壳可能为 1.2，小壳为 0.8。

### 5.4 密度（Density）

```
totalColorant = Σ colorant[型号]
density = normalize(totalColorant, 0, casing.capacity)
```

### 5.5 持续时间（Duration）

```
duration = normalize(thrust × 0.6 + totalEffect × 0.4, 0, MAX_DURATION_VALUE)
```

### 5.6 颜色向量（Color）

```
totalColorant = Σ colorant[型号]
for each color:
  color[colorName] = colorant[colorName] / totalColorant
```

### 5.7 特效向量（Effects）

```
for each effect agent:
  if amount >= threshold:
    effect[effectName] = (amount - threshold + 1) × intensityPerUnit
```

### 5.8 二次爆炸

仅当 casing.layers >= 2 且 effect[secondary] >= threshold 时触发：

```
secondary = {
  scale: scale × 0.6,
  density: density × 0.7,
  delay: 0.4 + fuse.lengthFactor × 0.3, // 秒
  color: 二层颜色剂比例,
  effects: 二层特效剂向量
}
```

### 5.9 成本

```
cost = casing.cost
       + Σ (gunpowder[型号] × unitCost)
       + Σ (colorant[型号] × unitCost)
       + Σ (effect[型号] × unitCost)
       + fuse.cost
```

## 6. 蓝图系统

### 6.1 蓝图数据结构

```js
{
  id: 'bp_xxx',
  name: '我的红牡丹',
  components: {
    gunpowder: { g001: 3 },
    casing: 'c001',
    colorant: { col001: 2 },
    fuse: 'f001',
    effect: { e001: 2 }
  },
  createdAt: 1718230000
}
```

### 6.2 蓝图槽位

- 初始槽位：5 个
- 扩展成本：每个 ¥100，逐次递增
- 上限：20 个（MVP）

### 6.3 保存/加载

- 玩家在组装台点击「保存蓝图」
- 输入名称（默认生成，如「红牡丹改」）
- 保存到 `state.blueprints`
- 蓝图列表显示在组装台「我的蓝图」区域
- 点击蓝图一键加载组件到组装槽

### 6.4 系统配方作为示例蓝图

- 系统配方不可删除
- 显示在「示例配方」区域
- 玩家可复制系统配方并修改后保存为新蓝图

## 7. 评分系统

### 7.1 事件偏好从离散改为连续

原事件偏好：

```js
preferences: { color: { red: 20 }, effect: { crackle: 15 }, height: { low: 10 } }
```

新事件偏好：

```js
preferences: {
  height: 0.7,        // 期望高度 0~1
  scale: 0.6,
  density: 0.5,
  duration: 0.6,
  color: { red: 0.6, gold: 0.4 },
  effects: { crackle: 0.8, tail: 0.3 },
  complexity: 0.4      // 对复杂设计的偏好
}
```

### 7.2 单发烟花匹配度

```
match = 
  1 - |shell.height - event.height|
  + 1 - |shell.scale - event.scale|
  + 1 - |shell.density - event.density|
  + 1 - |shell.duration - event.duration|
  + colorSimilarity(shell.color, event.color)
  + effectSimilarity(shell.effects, event.effects)
```

### 7.3 复杂度奖励

```
complexity = 0
if shell.hasSecondary: complexity += 0.15
if colorCount >= 2: complexity += 0.05
if effectCount >= 2: complexity += 0.05
```

### 7.4 重复惩罚

保留原有逻辑：同一烟花弹使用次数越多，惩罚越高。

### 7.5 最终得分

```
baseScore = average(shellMatch) / maxPossibleMatch × 100
score = baseScore × (1 + complexityBonus) × (1 - repeatPenalty)
score = clamp(score, 0, 100)
```

## 8. UI/UX 变更

### 8.1 组装台

- 左侧：五个组件类别面板
  - 火药：数量选择器（- / +）
  - 壳体：型号选择卡片
  - 颜色剂：多选数量选择器
  - 引线：型号选择卡片
  - 特效剂：多选数量选择器
- 中间：实时预览
  - 属性条：高度、规模、密度、持续时间
  - 容量条：当前占用 / 容量上限
  - 颜色预览圆点
  - 特效标签
  - 二次爆炸指示
- 右侧：节目单 + 蓝图列表

### 8.2 容量条

- 绿色：占用 < 70%
- 黄色：70% ~ 95%
- 红色：> 95% 或超容
- 超容时「加入节目单」和「保存蓝图」禁用

### 8.3 实时预览

- 组装变化时，右侧小预览图实时更新（用简化粒子或属性条）
- 完整表演仍用 Canvas 渲染

### 8.4 蓝图管理

- 「我的蓝图」列表显示名称、形状、成本
- 点击加载，长按/点击菜单删除
- 「示例配方」区域折叠/展开

## 11. 已确认决策

| 问题 | 决策 |
|---|---|
| 超容处理 | 禁止超容组装，无惩罚。UI 中容量条变红并禁用确认按钮 |
| 二层火药 | 不允许。二层只能装二次爆炸药、颜色剂、特效剂 |
| 实时预览 | 小 Canvas 演示，实时渲染当前组装效果 |
| 颜色混合 | 不限制颜色种类数，受颜色剂重量和壳体容量限制 |
| 特效叠加 | 无冲突规则，允许覆盖。多种特效同时生效 |
| 系统配方 | 精简数量，从 20 个减为 8~10 个示例蓝图 |

## 12. 组件描述与提示

每个组件在 UI 中必须显示：

- **名称**
- **简短描述**（如「最基础的火药，推力一般，便宜稳定」）
- **关键属性**（推力、容量、高度系数等）
- **解锁条件**（已解锁 / 需要 XX 名气）
- **单位成本**

组装台中，当玩家 hover 或聚焦某个组件时，显示完整描述 tooltip。
