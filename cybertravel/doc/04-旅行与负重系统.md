## 一、路线系统

### 路线数据结构

每条路线由多个**节点**组成，每个节点代表一个地理站点。

```javascript
节点 = {
  id: '城市英文标识',
  name: '中文名称',
  type: 'city' | 'wild',  // 城市 or 野外
  km: 累计里程,
  desc: '风景描述'
}
```

## 二、节点到达事件

到达每个城市节点时，自动触发该节点的描述事件，影响玩家属性：

| 节点 | 体力 | 精神 | 资金 | 素材 | 备注 |
|------|------|------|------|------|------|
| 成都 | - | - | - | - | 旅程起点，无效果 |
| 雅安 | +3 | +5 | -30 | - | 买雅鱼 |
| 泸定 | +3 | +3 | - | +3 | 拍摄铁索桥 |
| 康定 | -2 | - | - | +5 | 海拔2560m，轻微高反 |
| 新都桥 | +2 | +5 | - | +10 | 摄影天堂 |
| 理塘 | -5 | -3 | - | - | 海拔4014m，高反严重 |
| 巴塘 | +5 | +5 | -50 | - | 补充氧气药品 |
| 拉萨 | - | +20 | - | +15 | 抵达终点！ |

> 注：体力恢复仅在到达节点时触发一次。

## 三、旅行模式

### 两种赶路模式

| 模式 | 速度 | 精力消耗 | 体力消耗 | 时间 | 适用场景 |
|------|------|---------|---------|------|---------|
| **正常骑行** | 20km/h | -10 | -5 | +3h | 日常赶路，平衡消耗 |
| **拼命赶路** | 35km/h | -20 | -10 | +3h | 赶时间/体力充裕时 |

**每日可骑行次数**：
- 上午 1 次
- 下午 1 次
- 傍晚可选夜骑 1 次

**里程推进**：
- 每次骑行直接加上对应速度值（20 或 35）
- 如果累计里程 ≥ 下一节点里程，触发"到达节点"事件

## 四、负重系统

### 物品清单

| 物品 | 重量 | 用途 |
|------|------|------|
| 修车工具 | 2kg | 爆胎时可自行修理 |
| 口粮 | 1kg | 饥饿时恢复体力 |
| 帐篷 | 3kg | （预留）露营装备 |

### 初始装备

| 角色 | 修车工具 | 口粮 | 总重量 |
|------|---------|------|--------|
| 程序员 | 1（2kg） | 5（5kg） | 7kg |
| 狠活老铁 | 1（2kg） | 3（3kg） | 5kg |

### 负重规则

- **标准负重上限**：15kg
- **超载上限**：22.5kg（超载 50%）
- **超载惩罚**：每天夜间结算时，每超载 1kg → 体力 -2

```
超载量 = 当前重量 - 15
惩罚 = 超载量 × 2
```

**示例**：
- 背包 18kg → 超载 3kg → 每天体力额外 -6
- 背包 15kg → 不超载 → 无惩罚
- 背包 23kg → 无法拾取（超过 22.5kg 上限）

### 当前设计局限

> MVP 版本中，物品系统较为简单：
> - 口粮无法主动使用，仅在「饥饿事件」中被动消耗
> - 没有商店系统，无法购买新物品
> - 帐篷暂无实际用途（预留功能）
> 
> 未来可扩展：沿途商店、丢弃物品、物品交易等。
