# CyberTravel · 内容产出与平台算法

> 夜间内容创作、三平台算法、变现机制详解

---

## 一、内容类型

夜间阶段，玩家可以自由组合以下 **3 种内容类型**进行产出：

| 类型 | 名称 | 时间 | 精力 | 依赖属性 |
|------|------|------|------|---------|
| photo | 📷 摄影图文 | 1h | 10 | 审美（ART） |
| video | 🎬 剪视频 | 4h | 30 | 智力（INT） |
| live | 🔴 做直播 | 2h | 25 | 情商（HRT） |

**关键机制**：
- **可多选**：玩家可以同时选择多种类型组合发布（如 1 个视频 + 2 个直播）
- **资源约束**：总时间 ≤ 剩余时间（24 - 当前时间），总精力 ≤ 剩余精力
- **素材加成**：当前素材数量 × 2 作为质量加成（上限 50）

---

## 二、内容质量计算

### 单条内容质量公式

```
基础质量 = 50 + 角色对应属性值
素材加成 = min(素材数量 × 2, 50)
最终质量 = 基础质量 + 素材加成
```

**示例**（程序员，素材 10）：
- 剪视频：基础 = 50 + 85 = 135，素材加成 = 20，最终 = 155
- 摄影图文：基础 = 50 + 40 = 90，素材加成 = 20，最终 = 110

## 三、三平台算法

每个平台对不同内容类型有不同的**倍率偏好**：

| 内容类型 | B站 | 抖音 | 小红书 |
|---------|-----|------|--------|
| 摄影图文 | ×0.5 | ×0.3 | ×2.0 |
| 剪视频 | ×2.0 | ×1.0 | ×0.5 |
| 做直播 | ×0.8 | ×2.5 | ×0.2 |

### 粉丝增长公式

```
rawGain = floor((最终质量 + 属性加成 × 0.5) × 平台倍率 × 0.1)
属性加成 = 角色对应属性值

// 变现解锁加成
if 粉丝 ≥ 10万: monetizationBonus = 2.0
if 粉丝 ≥ 1万: monetizationBonus = 1.5
if 粉丝 ≥ 1千: monetizationBonus = 1.2
else: monetizationBonus = 1.0

最终增长 = floor(rawGain × monetizationBonus)
```

### 计算示例

**场景**：程序员，素材 10，发布 1 条视频

| 平台 | 计算过程 | 粉丝增长 |
|------|---------|---------|
| B站 | (155 + 42.5) × 2.0 × 0.1 × 1.2 | +47 |
| 抖音 | (155 + 42.5) × 1.0 × 0.1 × 1.0 | +19 |
| 小红书 | (155 + 42.5) × 0.5 × 0.1 × 1.0 | +9 |

> 程序员做视频在 B站 最有优势。

## 四、角色与平台匹配

| 角色 | 优势平台 | 优势内容 | 原因 |
|------|---------|---------|------|
| 程序员 💻 | **B站** | 剪视频 | 智力 85，B站视频倍率 ×2.0 |
| 狠活老铁 🛠️ | **抖音** | 做直播 | 情商 80，抖音直播倍率 ×2.5，且有翻倍天赋 |

**策略建议**：
- 程序员：白天多拍素材，晚上专注剪视频发 B站
- 老铁：白天拼命赶路积累狠活素材，晚上做抖音直播

## 五、变现机制

### 打赏收入

```
总粉丝增长 = B站增长 + 抖音增长 + 小红书增长
打赏收入 = floor(总粉丝增长 × 0.2)
```

**示例**：一晚涨粉 200 → 收入 ¥40

### 变现里程碑

| 平台 | 门槛 | 解锁能力 | 效果 |
|------|------|---------|------|
| B站 | 1万粉 | 创作激励 | 内容收益 ×1.5 |
| 抖音 | 5千粉 | 直播带货资格 | 内容收益 ×1.2（叠加） |

> 变现加成是乘法叠加的。同时解锁 B站和抖音 → 收益 ×1.5 ×1.2 = ×1.8

## 六、内容产出策略矩阵

| 策略 | 内容组合 | 适合角色 | 优点 | 缺点 |
|------|---------|---------|------|------|
| **精品视频流** | 只剪视频 | 程序员 | B站爆发高、粉丝质量好 | 耗时长（4h）、精力消耗大 |
| **直播冲量流** | 只开直播 | 老铁 | 抖音翻倍、互动感强 | 精力消耗高、素材利用率低 |
| **图文养老流** | 只发图文 | 均可 | 省时省力、可多平台铺量 | 粉丝增长慢 |
| **全能混合流** | 视频+直播+图文 | 高精力时 | 全平台覆盖、总收入最高 | 精力消耗极大 |
| **素材囤积流** | 白天疯狂拍素材，晚上发视频 | 程序员 | 素材加成拉满 | 白天无法赶路，进度慢 |

## 七、当前 MVP 局限

1. **没有内容库存**：当天素材不保留，每晚清空
2. **没有内容历史**：无法查看过去发布了什么
3. **没有粉丝互动反馈**：只有数字增长，没有评论/私信等反馈机制
4. **变现单一**：只有打赏，没有广告/带货/商单等多元收入

**未来扩展方向**：
- 内容库存系统（素材可累积多天）
- 粉丝评论系统（正反馈/负反馈影响 SAN）
- 品牌合作事件（粉丝达到一定数量后触发商单）
- 内容爆款回溯（某条视频突然二次传播）
