# CyberTravel · 骑行网红模拟器 — 总体设计

> 版本：MVP | 更新日期：2026-05-27

---

## 一、游戏定位

**CyberTravel** 是一款以"骑行川藏线 + 网红养成"为核心主题的**文字模拟经营游戏**。

玩家扮演一名骑行博主，从成都出发，沿 318 国道骑行至拉萨。旅途中需要平衡：
- **体能消耗**（体力、精力）
- **内容创作**（拍摄素材、剪辑视频、做直播）
- **粉丝增长**（B站、抖音、小红书三平台运营）
- **道德抉择**（真实记录 vs 编造剧本，影响 Sin 值）
- **资金管理**（食宿、修车、装备）

---

## 二、核心设计目标

| 目标 | 说明 |
|------|------|
| **沉浸感** | 通过真实川藏线地名、风景描述、高海拔反应等细节营造代入感 |
| **策略性** | 玩家需要在"赶路"和"创作"之间做权衡，不同的选择导向不同结局 |
| **道德张力** | "黑红也是红"的诱惑 vs 真实记录的坚守，形成核心冲突 |
| **轻量可玩** | 单 HTML 文件运行，无需后端，支持存档，单局 10-60 分钟 |

---

## 三、核心循环

```
晨间整备 → 日间行进（触发事件）→ 中午选择 → 下午行进 → 傍晚决策
    ↓
夜间内容产出（拍照/剪视频/直播）→ 结算（粉丝增长、收入、体力恢复）
    ↓
进入下一天 → ... → 抵达终点或中途死亡/崩溃/破产
```

---

## 四、技术架构

| 项目 | 说明 |
|------|------|
| **运行环境** | 纯前端，单个 HTML 文件，浏览器打开即可运行 |
| **状态管理** | `GameState` 类集中管理所有游戏数据 |
| **存档系统** | `localStorage` 自动存档，支持断点续玩 |
| **渲染引擎** | `Renderer` 类负责 DOM 更新和日志输出 |
| **事件引擎** | `EventEngine` 类按权重随机抽取事件 |
| **时间引擎** | `TimeEngine` 类管理游戏内时间推进和阶段切换 |

## 六、路线设计

### 短线（新手模式）
成都 → 雅安（140km）→ 泸定（220km）→ 康定（325km）

**约 4 个节点，游戏时长 10-15 分钟**

### 长线（标准模式）
成都 → 雅安 → 泸定 → 康定 → 新都桥（420km）→ 理塘（550km）→ 巴塘（680km）→ 拉萨（2100km）

**约 8 个节点，游戏时长 30-60 分钟**

## 八、结局概览

| 结局 | 类型 | 触发条件 |
|------|------|---------|
| 横死路边 | Bad | 体力 ≤ 0 |
| 精神崩溃 | Bad | 精神 ≤ 0 |
| 破产流浪 | Bad | 资金 ≤ 0 且 粉丝 < 1000 |
| 黑红之王 | Neutral | 黑红值 ≥ 80 或 抵达时 Sin ≥ 50 |
| 抵达圣城 | Neutral | 正常抵达拉萨 |
| 中国骑行第一人 | Good | 抵达时粉丝 > 10万 且 Sin < 20 |
