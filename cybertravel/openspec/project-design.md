# CyberTravelGame 产品设计文档

## 1. 产品定位

- **名称**: 骑行网红模拟器 (CyberTravelGame)
- **Slogan**: "在流量与生存之间，选择你的西行之路"
- **类型**: 文字生存 / 经营模拟 / 硬核 Roguelike
- **核心体验**: 时间驱动的生存与流量双轨博弈

## 2. 目标用户

- 核心: 18-35 岁，对骑行/户外/自媒体感兴趣的玩家
- 扩展: 喜欢文字冒险、经营模拟、社会讽刺题材的玩家
- 微信生态: 喜欢朋友圈分享、群互动的社交型玩家

## 3. 核心玩法设计 (经 grill-me 梳理确认)

### 3.1 时间驱动系统

游戏以真实时间进度为核心驱动，不再是固定四阶段：

- **早上 9:00**: 固定出发时间（可被晚上事件影响提早或延后）
- **白天 9:00-18:00**: 行进过程，时间进度条推进，路程逐渐靠近下一个目标点
- **中午/下午**: 玩家选择是否吃饭休息或赶路
- **晚上 19:00-24:00**: 内容产出时间，自由组合活动
- **18:00 后**: 可选择不扎营继续赶路（效率降低，消耗增加）

### 3.2 精力系统 (独立属性)

- 精力 = 行动力，决定白天能做多少事、晚上能产出多少内容
- 上限由体力(CON)和载具决定，**不会一觉补满**
- 恢复取决于睡眠事件：住宿类型（客栈 > 扎营）、海拔、装备
- 超载惩罚：背包超重后每天额外消耗体力

### 3.3 素材系统

- 白天事件自动积累素材（风景/冲突/困境/人文）
- 素材分类型存储，不同类型匹配不同平台偏好
- 晚上剪视频需要素材支撑，没素材则视频质量下降
- MVP: 统一数值池；正式版: 独立素材池

### 3.4 内容产出系统 (自由组合)

晚上 19:00-24:00 共 5 小时时间预算，自由组合：

| 形式 | 时间消耗 | 精力消耗 | 收益特征 |
|------|---------|---------|---------|
| 📷 摄影图文 | 1h | 低 | 涨粉慢但稳定，适合接广告 |
| 🎬 剪视频 | 3-4h | 高 | 粉丝粘性最强，长尾效应 |
| 🔴 做直播 | 2h | 高 | 即时打赏现金流之王 |

- 可同时做多个（如直播2h + 图文1h），但精力消耗叠加
- 内容质量取决于：对应属性 + 器材 + 素材积累

### 3.5 三平台差异化算法

| 平台 | 偏好 | 加成机制 | 惩罚机制 | 变现解锁 |
|------|------|---------|---------|---------|
| 弹幕网 (B站) | 硬核、纯粹、反资本 | 高质量视频指数裂变 | 造假被扒皮，精神值雪崩 | 1万粉: 创作激励 |
| 短视频网 (抖音) | 冲突、反转、狠活 | 直播/短视频爆发力极强 | 不整活瞬间取关 | 5千粉: 直播带货 |
| 种草网 (小红书) | 审美至上、精致 | 摄影曝光率高，广告稳定 | 审美不足无流量 | 1千粉: 品牌置换 |

- 低门槛变现：自动收益（每天根据粉丝数结算）
- 高门槛变现：主动事件（品牌合作/大额赞助），可带 Sin 抉择

### 3.6 身份选择系统 (MVP: 2个角色)

| 身份 | 核心优势 | 核心劣势 | 初始资金 | 推荐流派 |
|------|---------|---------|---------|---------|
| 前大厂程序员 | 视频爆款率+30% | 荒野精神值额外-10% | ¥25,000 | 中长视频 |
| 民间狠活老铁 | 黑红Sin减半，短视频翻倍 | 审美极低 | ¥500 | 直播/狠活 |

正式版扩展至4个角色（+ MCN破产小美、伪骑行大V）。

### 3.7 路线系统 (MVP: 2条路线)

| 路线 | 长度 | 节点数 | 预计时长 | 定位 |
|------|------|--------|---------|------|
| 短线 | 成都→康定 | 4个 | 10-15分钟 | 新手模式，资源宽松 |
| 长线 | 成都→拉萨 | 8个 | 30-60分钟 | 标准模式，完整体验 |

节点类型：城镇（可补给/休整/取快递）/ 荒野（只能扎营）。

### 3.8 事件系统 (MVP: 23个事件)

| 类型 | 数量 | 触发方式 | 例子 |
|------|------|---------|------|
| 大事件 | 5-8个 | 里程碑+概率 | 爆胎、遇到NPC、商单邀约 |
| 小事件 | 10-15个 | 概率触发 | 天气变化、路人合影、小修小补 |
| 节点事件 | 8个 | 到达触发 | 每个节点的特殊剧情 |

事件密度：每2-3天1个大事件，其他天小事件。

### 3.9 装备/消耗品系统 (MVP: 极简)

- 角色开局自带固定装备
- 城镇可购买消耗品：修车工具、食物/口粮
- 背包负重限制（固定上限15kg），可超载（每天额外扣CON）
- 正式版扩展：装备升级、载具改造、网购系统

### 3.10 原罪(Sin)系统 (MVP: 弱信号)

- Sin是隐藏属性，玩家看不到具体数值
- 来源：接虚假宣传商单、编造剧本、破坏环境
- 弱信号反馈：评论区偶尔出现质疑、粉丝增长放缓
- 正式版扩展：正义打击、全网封杀、行政拘留等结局

### 3.11 结局系统 (MVP: 5个结局)

| 结局 | 类型 | 触发条件 |
|------|------|---------|
| 《横死路边》 | bad | 体力 ≤ 0 |
| 《精神崩溃》 | bad | SAN ≤ 0 |
| 《破产流浪》 | bad | 资金 ≤ 0 且 粉丝 < 1000 |
| 《抵达圣城》 | neutral | 到达拉萨（保底） |
| 《骑行第一人》 | good | 到达拉萨 + 总粉丝 > 10万 + Sin < 20 |

结局配置化设计，支持后续扩展。条件冲突时按 priority 优先触发。

## 4. 界面设计

### 4.1 视觉风格

- **主题**: 赛博朋克 × 西部荒野
- **主色调**: 深色背景 (#0a0a0f) + 霓虹青 (#00f0ff) + 危险红 (#ff0055)
- **字体**: 系统默认中文字体，等宽数字显示

### 4.2 核心界面

**状态面板** (常驻顶部):
```
┌─────────────────────────────┐
│ 骑行网红模拟器      09:30    │
├─────────────────────────────┤
│ 体力 ████████░░  精力 ██████░░│
│ 资金 ¥21,000     粉丝 1.2K  │
│ 公里 420km → 理塘(130km)    │
│ 负重 12/15kg                │
└─────────────────────────────┘
```

**场景/事件区** (居中):
- 打字机效果显示文本（可点击跳过）
- 时间进度条显示当前时间

**操作区** (底部):
- 选项按钮（纯点击，移动端友好）
- 数量选择用按钮组（"买1个/买5个/不买"）

## 5. 信息透明度

- **显示**: 体力、智力、审美、情商、精神、资金、粉丝、精力、负重
- **隐藏**: 运气(Luck)、原罪(Sin)
- **明示**: 三平台算法偏好直接写在界面上

## 6. 存档系统

- MVP: 单槽位，localStorage，存 schemaVersion 占位
- 自动存档：每天结算后
- 手动存档：玩家主动保存
- 正式版: 多槽位 + 版本迁移 + 服务器同步

## 7. 微信生态特性 (正式版)

1. **朋友圈通报海报**: 游戏结束时生成"社会新闻"风格海报
2. **群友赞助机制**: 濒死时向微信群发起众筹，观看激励广告
3. **好友排行榜**: 骑行里程/粉丝数排行
4. **每日挑战**: 每日刷新特殊挑战条件

## 8. MVP 范围总结

- [ ] 三平台差异化算法
- [ ] 2个角色（程序员、狠活老铁）
- [ ] 2条路线（短线新手/长线标准）
- [ ] 23个事件（5大+10小+8节点）
- [ ] 极简装备+消耗品+负重
