# ADR-001: 单文件 MVP 架构

## 状态
superseded by [ADR-006: 模块化 ESM 重构](./ADR-006-modular-esm-refactor.md)

## 背景
CyberTravelGame 需要快速验证核心玩法。选择单文件架构可以降低开发门槛、便于分享、无需构建工具。

## 决策
MVP 阶段采用单 HTML 文件架构：
- 所有 HTML/CSS/JS 集中在 index.html
- 无外部依赖
- 浏览器直接打开即可运行
- 数据配置以内联 JS 对象形式存在

## 后果

### 正面影响
- 零构建配置，即开即写
- 便于快速原型验证
- 易于分享给非技术用户测试
- 无网络依赖，离线可玩

### 负面影响
- 代码组织受限，难以模块化管理
- 代码体积增长后维护困难
- 无法使用现代 JS 模块化特性
- 测试困难

### 缓解措施
- 使用 IIFE (立即执行函数) 模拟模块隔离
- CSS 使用 BEM-like 命名规范避免冲突
- MVP 验证后 (Phase 3) 迁移到模块化架构
