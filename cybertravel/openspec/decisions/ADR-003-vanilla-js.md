# ADR-003: 纯原生 JS 无框架

## 状态
accepted

## 背景
MVP 需要最小化依赖，最大化可移植性。引入前端框架会增加构建复杂度和包体积。

## 决策
MVP 阶段使用纯原生 JavaScript (ES6+)，不引入任何前端框架或库。

## 后果

### 正面影响
- 零依赖，零构建
- 最小包体积
- 最大浏览器兼容性
- 无框架学习成本

### 负面影响
- 手动 DOM 操作繁琐
- 状态管理需自行实现
- 无组件复用机制
- 代码组织受限

### 缓解措施
- 封装简易的 DOM 操作工具函数
- 使用原生 Class 组织代码
- 正式版 (Phase 3) 迁移到小程序框架
