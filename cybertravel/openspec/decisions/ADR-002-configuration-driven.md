# ADR-002: 配置驱动设计

## 状态
accepted

## 背景
游戏需要丰富的内容（事件、角色、路线、道具）。硬编码内容会导致代码与数据耦合，不利于内容迭代和扩展。

## 决策
采用配置驱动设计：
- 所有游戏内容通过 JSON/JS 对象配置定义
- 引擎负责解析配置并驱动游戏流程
- 内容创作者无需修改引擎代码即可添加新内容

## 后果

### 正面影响
- 内容与逻辑解耦，便于并行开发
- 非程序员可以参与内容创作
- 便于 A/B 测试和动态调整
- 正式版可扩展为服务器端配置热更新

### 负面影响
- 需要设计完善的配置 Schema
- 配置错误可能导致运行时问题
- 需要配置验证机制

### 缓解措施
- 定义严格的配置 Schema 和验证函数
- MVP 阶段使用内联配置，正式版分离为独立 JSON 文件
- 引擎启动时进行配置完整性检查
