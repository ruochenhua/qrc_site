# MVP HTML 单文件实现

## 背景与动机

CyberTravelGame 的核心玩法需要快速验证。当前项目已有：
- 完整的游戏设计文档 (mvp-design.md)
- 初步的 HTML 原型 (index.html，约 800+ 行)

但现有原型功能不完整，缺少：
- 完整的四阶段循环
- 事件系统
- 内容产出与平台算法
- 结局判定
- 存档系统

本 Change 旨在完成一个**可玩的 MVP 版本**。

## 范围

### In Scope
- 单文件 HTML 架构 (ADR-001)
- 游戏状态机与核心数据结构
- 四阶段循环 (晨间→日间→夜间产出→夜间结算)
- 事件引擎与基础事件库 (10-15 个事件)
- 三平台粉丝系统 (简化算法)
- 内容产出系统 (摄影/视频/直播)
- 判定与结算系统
- 基础结局系统 (3-5 个结局)
- localStorage 存档 (ADR-005)
- 赛博朋克风格 UI

### Out of Scope
- 多角色选择 (仅实现程序员角色)
- 完整路线 (仅 5-8 个节点)
- 载具损耗详细系统
- 物流/网购系统
- 动态平台风向
- 朋友圈海报生成
- 微信生态功能

## 影响

- **文件**: index.html (主要修改)
- **新增**: openspec/changes/mvp-html-implementation/ (本 Change 文档)
- **无外部依赖**: 纯原生 JS/CSS

## 用户故事

### Story 1: 完成一局游戏
**作为** 玩家，**我希望** 能从开局玩到结局，**以便** 体验完整的核心循环。

**验收标准**:
- [ ] 可以创建新游戏
- [ ] 可以经历每日四阶段
- [ ] 可以到达终点或触发 Game Over
- [ ] 可以看到结局画面

### Story 2: 做出策略选择
**作为** 玩家，**我希望** 在事件中做出不同选择并看到后果，**以便** 感受策略深度。

**验收标准**:
- [ ] 事件提供多个选项
- [ ] 不同选项有不同后果
- [ ] 属性影响判定结果

### Story 3: 内容产出与涨粉
**作为** 玩家，**我希望** 选择内容形式并看到粉丝增长，**以便** 体验流量博弈。

**验收标准**:
- [ ] 可选择摄影/视频/直播
- [ ] 不同形式有不同收益
- [ ] 粉丝数实时更新

### Story 4: 存档与继续
**作为** 玩家，**我希望** 保存和加载游戏进度，**以便** 分多次游玩。

**验收标准**:
- [ ] 自动存档功能
- [ ] 手动存档功能
- [ ] 加载存档后继续游戏
