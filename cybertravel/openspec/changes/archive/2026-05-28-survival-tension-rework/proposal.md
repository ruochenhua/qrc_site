# 生存张力重构（Survival Tension Rework）

## 背景与动机

当前 MVP 的核心循环存在两个结构性问题：

1. **生存压力不足**：程序员角色（CON=35）正常骑行一天净体力变化约为 +1，只要不触发极端事件就能无限苟活。生存线没有形成"每天都在为活着焦虑"的张力。
2. **流量压力缺失**：玩家选择"稳妥骑行+随便发内容"可以平安抵达终点，但不会有任何负面后果。文档中设想的"生存线与流量线的反复撕扯"完全没有出现——既没有"不进则退"的被动压力，也没有"冒险爆红"的主动诱惑。

本 Change 通过重构事件系统、引入四季天气系统、扩展路线节点、建立内容质量衰减机制，把核心矛盾从"事件抽奖暴毙"转变为**"苟活能保命但流量会死，冒险能爆红但肉体可能会死"的持续博弈**。

## 范围

### In Scope
- 事件系统三分法重构（安全 / 冒险 / 黑红）
- 四季开局选择（春/夏/秋/冬）
- 每日天气系统（晨间播报 + 全局 Buff/Debuff）
- 路线节点扩展（8 节点 → 22 节点，新增 town / outpost / wild_poi 类型）
- 内容质量硬门槛机制（<60 分掉粉 + SAN 惩罚）
- 素材保质期机制（3 天，过期清零）
- 连续低质量惩罚（"过气" Debuff）
- 顶部天气状态栏 + 日志叙事播报
- 程序员专属惩罚落地（荒野 SAN 额外消耗）

### Out of Scope
- 多角色扩展（仍只有程序员 + 老铁）
- 载具损耗与抛锚决策树
- 物流/网购系统
- Sin 正义打击完整机制（保持弱信号）
- 独立素材池分类型（仍用统一数值池）
- 动态平台风向事件

## 影响

- **文件**: `cybertravel/index.html`
- **数据变更**:
  - `ROUTE` 重构为 22 节点（短线 8 节点 / 长线 22 节点）
  - `EVENTS` 全部事件按三分法重写
  - 新增 `SEASONS` 配置表
  - 新增 `WEATHER_TYPES` 配置表
  - `GameState` 增加 `season`, `weather`, `materialsDay`（素材采集日戳）, `lowQualityStreak`
- **系统交互**:
  - 与夜间结算系统耦合（内容质量判定 + 保质期衰减 + 过气 Debuff）
  - 与事件系统耦合（天气 Buff 影响事件选项数值）
  - 与兴趣点系统耦合（outpost 在极端天气下的生存意义）
- **存档变更**: schemaVersion 3 → 4（新增字段 + 路线结构变更）

## 用户故事

### Story 1: 选择季节开局
**作为** 玩家，**我希望** 开局时选择季节，**以便** 体验完全不同的生存节奏。

**验收标准**:
- [ ] 开局流程：选路线 → 选角色 → 选季节
- [ ] 四季有明显差异化参数（事件频率、体力消耗、素材加成）
- [ ] 冬季程序员生存难度显著高于春季

### Story 2: 每天收到天气播报
**作为** 玩家，**我希望** 每天早晨知道今天的天气和全局影响，**以便** 做出策略决策。

**验收标准**:
- [ ] 顶部状态栏常驻显示天气图标和关键 Buff
- [ ] 晨间日志自动打印天气叙事文本
- [ ] 天气影响全天数值（体力消耗、素材品质、事件危险度）

### Story 3: 事件中面临安全与冒险的选择
**作为** 玩家，**我希望** 每个事件都有明确的"安全稳"和"冒险赚"选项，**以便** 根据当前状态做赌局决策。

**验收标准**:
- [ ] 每个大事件至少包含：安全选项 + 冒险选项
- [ ] 合适的事件增加黑红选项（Sin 换流量）
- [ ] 冒险选项的收益显著高于安全选项
- [ ] 黑红选项的收益显著高于冒险选项，但有 Sin 代价

### Story 4: 内容质量不及格导致掉粉
**作为** 玩家，**我希望** 晚上随便发低质量内容会受到惩罚，**以便** 有动力去冒险攒素材。

**验收标准**:
- [ ] 内容产出计算质量分，60 分为及格线
- [ ] <60 分时所有平台掉粉，SAN -3
- [ ] 连续 2 天不及格触发"过气" Debuff（3 天质量 ×0.7）
- [ ] 质量分在日志中明示给玩家

### Story 5: 素材有保质期压力
**作为** 玩家，**我希望** 素材不能无限囤积，**以便** 规划"攒素材→爆发产出"的节奏。

**验收标准**:
- [ ] 素材保质期 3 天（采集当日 + 后两天）
- [ ] 过期素材在夜间结算时自动清零
- [ ] 日志提示"XX 份素材已过期失效"

### Story 6: 在 outpost 绝境求生
**作为** 玩家，**我希望** 在到达大城之前能在路边驿站紧急补给，**以便** 在冬季长路段存活。

**验收标准**:
- [ ] 路线新增 town（简化城市）和 outpost（路边驿站）节点
- [ ] outpost 只有基础商品，无高级住宿
- [ ] 到达 outpost 时显示补给界面
