# 兴趣点系统（POI System）

## 背景与动机

当前游戏中玩家只能看到抽象的"里程数字"，无法直观感知自己在川藏线上的具体位置和进度。到达节点时虽然有文本描述，但缺少：
- **空间感知**：不知道自己在哪、下一站多远
- **探索乐趣**：路过著名景点却无法互动
- **策略深度**：没有补给决策、住宿选择的资源管理维度
- **进度可视化**：无法查看全程路线和当前位置

本 Change 引入**兴趣点系统（POI System）**，将抽象路线转化为可探索、可交互的空间节点，增强沉浸感和策略深度。

## 范围

### In Scope
- 兴趣点分类体系（城市 / 野外景点 / 普通路段）
- 城市商店系统（购买口粮、氧气、修车工具等）
- 城市住宿系统（旅馆/酒店/豪华酒店，差异化恢复效果）
- 景点参观系统（城市景点 + 野外景点，获取素材和精神恢复）
- 野外扎营机制（未到达城市时强制露营，恢复缩减 + 夜间突发事件）
- 全程地图 UI（常驻距离提示 + 可展开的路线可视化）
- 当前阶段与下一兴趣点距离常驻显示

### Out of Scope
- 多路线切换（仍使用现有短/长路线）
- 交通工具更换（仍只有自行车）
- NPC 交互对话系统
- 天气系统对兴趣点的动态影响
- 兴趣点的随机生成（全部预配置）

## 影响

- **文件**: `cybertravel/index.html`
- **数据变更**: `ROUTE` 节点数据结构扩展（增加 `type`、`shop`、`attractions`、`accommodation` 字段）
- **UI 变更**: 顶部常驻信息栏、全程地图弹窗、城市交互菜单
- **系统交互**: 
  - 与夜间结算系统耦合（住宿/露营影响恢复公式）
  - 与事件系统耦合（野外夜间增加突发事件池）
  - 与负重系统耦合（购买物品增加重量）

## 用户故事

### Story 1: 查看当前进度和全程地图
**作为** 玩家，**我希望** 在游戏界面上看到当前位置和距离下一个兴趣点的距离，**以便** 了解自己的行程进度。

**验收标准**:
- [ ] 游戏界面上常驻显示"距离下一个兴趣点：XX km"
- [ ] 点击可展开全程地图弹窗
- [ ] 地图上标注当前位置、已到达节点、未到达节点
- [ ] 地图显示各节点名称、类型图标、累计里程

### Story 2: 到达城市进行补给和住宿
**作为** 玩家，**我希望** 到达城市时可以购买补给、选择住宿档次，**以便** 管理资源和恢复状态。

**验收标准**:
- [ ] 到达城市后显示"进入城市"按钮
- [ ] 打开城市菜单：商店 / 住宿 / 景点 / 离开
- [ ] 商店可购买口粮、氧气瓶、修车工具（资金、负重检查）
- [ ] 住宿分三档，价格和恢复效果不同
- [ ] 参观景点消耗时间，获得素材和精神恢复

### Story 3: 路过野外景点拍摄素材
**作为** 玩家，**我希望** 路过野外著名景点时可以选择停下来拍摄，**以便** 获得大量优质素材。

**验收标准**:
- [ ] 到达野外景点时触发到达事件
- [ ] 可选择"停留拍摄"或"路过"
- [ ] 拍摄获得大量素材（10-20）和精神恢复（5-15）
- [ ] 消耗 0.5-1 小时时间

### Story 4: 野外扎营与夜间突发事件
**作为** 玩家，**我希望** 晚上未到达城市时只能在野外扎营，**以便** 体验旅途的不确定性。

**验收标准**:
- [ ] 傍晚判断当前是否在 city 类型节点
- [ ] 不在城市 → 只能"野外扎营"，无法购买/住宿
- [ ] 野外扎营恢复系数为城市的 60%
- [ ] 野外过夜 20% 概率触发夜间突发事件

### Story 5: 查看兴趣点详情
**作为** 玩家，**我希望** 在全程地图上点击任意节点查看详情，**以便** 提前规划行程。

**验收标准**:
- [ ] 地图上每个节点可点击
- [ ] 弹出详情面板：名称、类型、描述、距离当前位置
- [ ] 城市节点显示可购买商品列表
- [ ] 景点节点显示可拍摄内容
