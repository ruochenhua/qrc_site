# MVP 核心循环验证

## 背景与动机

经过 grill-me 玩法梳理，CyberTravelGame 的核心玩法已明确：
- 时间驱动系统（9:00-24:00 时间进度条）
- 精力系统（独立属性，睡眠恢复不补满）
- 素材积累 + 内容产出自由组合
- 三平台差异化算法

本 Change 旨在验证**最核心的一条体验路径**：玩家从开局到第一个节点，完整经历"白天赶路→事件→晚上产出→结算→第二天"的循环。

## 范围

### In Scope
- 单文件 HTML 架构 (ADR-001)
- 游戏状态机（含时间、精力、资金、粉丝）
- 时间驱动器（推进时间、阶段切换）
- 极简路线（2个节点：成都→雅安）
- 1个角色（程序员）
- 1个大事件 + 2个小事件
- 内容产出系统（单选：摄影/视频/直播）
- 三平台简化算法
- 基础 UI（状态面板 + 场景文本 + 选项按钮）
- localStorage 存档

### Out of Scope
- 负重系统
- 素材系统（视频质量统一）
- Sin 系统
- 结局系统（到达终点即结束）
- 消耗品购买
- 自由组合内容产出（MVP 先单选）
- 多角色/多路线

## 影响

- **文件**: index.html
- **无外部依赖**

## 用户故事

### Story 1: 完成第一天的核心循环
**作为** 玩家，**我希望** 从早上9点出发，经历白天事件，晚上选择内容产出，看到粉丝和资金变化，**以便** 理解游戏核心玩法。

**验收标准**:
- [ ] 可以看到时间从9:00推进到24:00
- [ ] 白天至少触发1个事件
- [ ] 晚上可以选择一种内容产出
- [ ] 结算后看到粉丝和资金变化
- [ ] 可以进入第二天

### Story 2: 到达第一个节点
**作为** 玩家，**我希望** 推进到雅安并触发到达事件，**以便** 体验节点机制。

**验收标准**:
- [ ] 推进约140km后到达雅安
- [ ] 触发节点到达事件
- [ ] 可以选择休整或继续前进
