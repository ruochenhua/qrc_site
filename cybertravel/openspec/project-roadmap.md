## Phase 0: MVP (当前)

**目标**: 一个可玩的单文件 HTML 版本，验证核心玩法循环

**完成标准**:
- [ ] 玩家可以完成一局游戏（从开局到结局）
- [ ] 核心四阶段循环完整运行
- [ ] 至少 3 种不同结局可触发
- [ ] 无阻塞性 Bug

**任务分解** (见 changes/mvp-html-implementation/tasks.md):

| 任务 | 模块 | 预估 |
|------|------|------|
| T01: 游戏状态机与数据结构 | engine/state.js | 1h |
| T02: 阶段管理器 | engine/phase.js | 1.5h |
| T03: 事件引擎 | engine/event.js | 1.5h |
| T04: UI 渲染系统 | ui/renderer.js | 2h |
| T05: 内容产出系统 | engine/content.js | 1h |
| T06: 平台算法 | engine/platform.js | 1h |
| T07: 判定与结算 | engine/judge.js | 1h |
| T08: 存档系统 | engine/save.js | 0.5h |
| T09: 事件配置数据 | data/events.json | 1h |
| T10: 路线节点配置 | data/route.json | 0.5h |
| T11: 结局系统 | engine/ending.js | 1h |
| T12: 整合与测试 | index.html | 2h |

## Phase 1: 内容扩展

**目标**: 丰富游戏内容，增加重玩价值

- 多角色选择（4 个完整身份）
- 事件库扩展至 50+ 个事件
- 路线扩展（川藏线完整 15+ 节点）
- 装备系统（购买/维修/损耗）
- 动态平台风向事件

---

## Phase 2: 系统深化

**目标**: 完善核心系统深度

- 载具损耗与抛锚决策树
- 物流时滞与网购系统
- NPC 偶遇与交互系统
- 多周目继承机制

---

## Phase 3: 平台迁移

**目标**: 迁移至微信小程序

- WXML/WXSS/JS 重构
- 微信 API 接入（分享、支付、广告）
- UI 适配移动端
- 服务器端存档同步

---

## Phase 4: 社交与变现

**目标**: 社交裂变与商业闭环

- 朋友圈通报海报生成
- 群友赞助 + 激励广告
- 好友排行榜
- 每日挑战活动
