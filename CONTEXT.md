# QrcSite — 领域上下文

## 项目定位

QRC-Eye 个人创作者主页 + 想法试验场。静态站点，GitHub Pages 托管。

站点由你（主创作者）维护，朋友作为间歇性合作者参与个别项目。

## 核心领域语言

| 术语 | 含义 |
|------|------|
| **QRC-Eye** | 你的个人创作者厂牌/标签，不是设计工作室，也不接商业委托 |
| **想法试验场** | 尝试各种产品原型和点子的地方，目前以小游戏为主，但不局限于此 |
| **原型 / 实验性产品** | 轻量可交互的原型作品，目前多为 HTML5 小游戏（如 CyberTravel），未来可能包含其他形式 |
| **个人 IP 定位** | 优先是「把奇怪想法快速做成可交互原型的人」；其次是「还在寻找自己表达方式的创作者」；再次才带一点社会观察/讽刺气质 |
| **CyberTravel** | 骑行网红模拟器 HTML5 游戏，部署在 `/cybertravel/` |
| **烟花大师 / Firework Master** | 烟花表演经营模拟 HTML5 游戏，玩家以烟花学徒身份参赛、解锁材料、自由组装烟花、积累名气 |
| **组件 / Component** | 可解锁的基础材料，分为火药、壳体、颜色剂、引线、特效剂五类。每类包含多种型号，玩家按数量添加 |
| **烟花弹 / Shell** | 玩家把一组组件按规则组装后得到的单发烟花，是表演的最小单位 |
| **配方 / Recipe** | 系统自带的固定蓝图/示例设计，随名气解锁后需消耗资金「研发」购买，购买后可在比赛中直接使用 |
| **蓝图 / Blueprint** | 玩家自创并保存的组件组合，可一键加载复用。有数量上限，可消耗资金扩展 |
| **壳体容量 / Casing Capacity** | 壳体能容纳的组件总量上限。火药、颜色剂、特效剂、二次装药的总占用不能超过容量 |
| **多层壳 / Multi-break Casing** | 支持二次爆炸的特殊壳体，内部有独立的主层和二层容量分区，由延时引线连接 |
| **比赛** | 《烟花大师》中以名气收益为主、资金收益为辅的事件。通常带有预算或其他硬性限制 |
| **活动** | 《烟花大师》中以资金收益为主、名气收益为辅的事件。预算由玩家自行规划，限制更宽松 |
| **表演 / 节目单** | 玩家为某场事件组装的一组烟花弹集合，系统据此实时渲染烟花表演并评分 |
| **资金** | 《烟花大师》中的主要消耗货币，用于购买材料、支付参赛成本、扩展蓝图槽位 |
| **名气** | 《烟花大师》中的声望指标，解锁更高级的比赛/活动与更稀有的组件 |
| **解锁** | 某个组件因名气达到门槛而出现在可购买列表中 |
| **研发** | 在配方实验室中支付资金，将已解锁的系统配方永久学会，使其可在比赛中直接使用 |
| **晋升** | 完成当前等级所有主线赛事且名气达到阈值后，系统自动晋升至下一职业等级，并以 Toast 提示玩家 |
| **评分公式** | 连续值匹配：事件对高度、规模、颜色、持续时间、特效等维度设有连续偏好，玩家节目单的总分由各烟花属性与事件偏好的拟合度计算，并辅以复杂度奖励与重复惩罚 |
| **预算规则** | 比赛采用硬预算：超出预算则无法参赛。活动采用软预算：允许超支，但收益可能无法覆盖成本，导致亏损 |
| **职业等级** | 玩家的成长阶段，从低到高为：学徒、熟手、技师、名家、大师。每级对应可参加的赛事等级，并解锁新的组件。MVP 以达成大师为通关条件 |
| **主线赛事** | 每个职业等级必须完成的固定赛事，完成后方可晋升到下一等级。首次通关有额外奖励 |
| **重复事件** | 随机刷新的可重复参加的事件（活动或比赛），供玩家刷取资金、名气等资源 |
| **烟花表演渲染** | 使用 Canvas 2D 粒子系统实时呈现烟花升空、爆炸、拖尾、二次爆炸等效果，带有随机性和物理拟真 |
| **保底事件** | 收益稳定、难度极低的可重复事件，确保玩家即使资金紧张也能持续获得少量资金，避免游戏卡死或硬性失败 |
| **Firework Master MVP** | 《烟花大师》最小可玩版本：5 等级、自由组件组装、系统配方示例、玩家蓝图保存、15 个事件、Canvas 粒子表演、双货币成长、localStorage 存档。部署在 `/firework-master/`
| **OpenSpec** | 配置驱动的变更管理流程 |
| **MVP** | 单文件 HTML5 游戏，最小可玩版本 |

## 项目结构

```
qrcsite/
├── index.html          # 主页
├── cybertravel/        # 游戏目录
│   ├── index.html      # 游戏 DOM 容器
│   ├── style.css       # 游戏样式
│   ├── js/             # 游戏逻辑（ES Module）
│   │   ├── config.js   # 配置数据（路线/事件/物品/天气等）
│   │   ├── systems.js  # 逻辑系统（时间/事件/内容/平台/黑红）
│   │   ├── state.js    # 状态机 + 存档系统
│   │   ├── renderer.js # UI 渲染器
│   │   └── game.js     # 游戏主控 + 入口
│   ├── openspec/       # 变更管理
│   └── doc/            # 设计文档
├── js/                 # 网站 JS
└── dev-blog/           # 实验笔记（当前未在首页导航展示）
```

## 架构约束

- 纯静态站点，无后端
- GitHub Pages 自动部署
- 游戏数据通过 Git 版本管理
